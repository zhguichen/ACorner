# ACorner 开发指南

## 项目定位

一隅 ACorner 是一个 macOS 原生悬浮单任务计时器。它的体验目标是低干扰而非效率监督：默认只显示圆点，时间到后不强制打断用户，所有记录由用户选择的本地文件夹保存。

## 目录

```text
Sources/ACorner/
  App/                 应用生命周期、设置入口
  Domain/              无 UI 的任务状态与持久化模型
  Features/Timer/      计时状态机、悬浮面板和 SwiftUI 卡片
  Services/            本地记录文件和保存目录选择
Tests/ACornerTests/    单元测试
docs/                  产品、架构、测试与迭代文档
docs/reference/        保留的原始产品与开发说明
```

## 开发环境与验证

最低目标为 macOS 14、Swift 6。每次修改后至少运行：

```sh
swift build
swift test
```

涉及交互、窗口、存储或计时逻辑时，再按 `docs/test.md` 完成相关人工验收。不要把 `.build/`、用户选择的记录文件夹或本机偏好数据加入版本控制。

## 修改纪律

- 保持 `TaskSessionModel` 负责计时状态机，`RecordStore` 负责文件系统；不要把 AppKit 文件对话框塞进 SwiftUI 卡片。
- 计时必须基于 `Date` 的真实时间差，不能依赖界面刷新次数。
- 浮窗自动展开不得抢键盘焦点；只有用户主动输入时才可激活应用。
- 颜色必须始终搭配状态文字；不得只用颜色传达状态。
- 不要增加暂停、倒计时秒数、提示音、效率评分、云端传输或后台监控，除非需求明确变更。
- 不要在未被要求时加入回退路径、兼容分支或替代数据源。
- 改动产品边界或架构时同步更新 `docs/prd.md`、`docs/rfc.md`；有意义的实现决策追加到 `docs/working.md`；改变验收方式则更新 `docs/test.md`。

## 隐私与安全

任务标题与备注属于用户数据。只可写入用户在原生文件选择器中明确选定的目录；不得上传、分析、埋点、读取屏幕内容或接入第三方服务。日志与错误信息中避免打印任务正文或备注。
