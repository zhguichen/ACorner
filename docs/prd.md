# PRD：一隅 ACorner

## 问题与目标

持续可见的倒计时、效率评价和强提醒会打断本已进入状态的用户。一隅只帮助用户为当前一件事留出一段时间：时间存在，但不持续催促。

目标用户是希望在 macOS 桌面上安静推进一件事、又不希望使用完整任务管理工具的人。

成功标准是用户可以从灰色细胶囊条开始一项任务，在不看秒数的情况下自然结束或转入下一项；每天凌晨 3 点后，先回顾昨天并确认今天的待办（也可选择今天留白），再进入完整功能；期限事项每天都清晰可见，到期当天显示文字状态但不打断用户；计时中和结束后都能在同一个记录框里写备注；胶囊条内的进度线会随着当天待办完成度逐步填满；完成记录、每日待办与期限事项都能可靠地保存在自己选择的本地文件夹，重启后不会丢失正在进行的任务、记录草稿或标题草稿。

## 核心流程

1. 启动后若当天尚未确认，或每天凌晨 3 点刷新后，展开面板先进入 Daybreak：回顾 Yesterday’s Echo 中昨天已完成与未完成的待办；未完成项可选择带到今天，也可确认今天留白。确认前不显示计时、常规待办或设置功能。
2. 确认当天后显示可拖动的细胶囊条，胶囊条分两行显示：第一行显示状态或当前任务，第二行显示当天待办完成度线。
3. 展开面板，输入目标并键入 1 到 180 分钟的投入时长。
4. 用户可从任意浮窗卡片的“设置”按钮打开设置窗口，维护当天待办与已完成事项；设置窗口也保留回到主面板的按钮。
5. 开始后长条变绿，默认不展示任务和时间；展开后可以在较大的记录框里输入过程记录。
6. 到计划结束时长条变黄并展开；十秒未操作则变蓝并进入正计时。
7. 用户点击“结束”后进入收尾状态，继续编辑同一个记录框。
8. 用户点击“Next Mission”后保存记录、勾选关联待办，并回到下一任务输入。
9. 用户可在设置中的“期限”页记录一次性日期事项；它们每天显示在 Daybreak 和期限页中，到期当天显示“今天截止”，过期未完成则显示“仍待处理”，但不弹窗、响铃或抢焦点。
10. 设置中的待办页在“已完成”之后列出过去每天未完成的待办，用户可选择“加入今天”或删除整条未完成任务脉络；每次带入都保留同一个任务脉络，带入今天后立即从过去未完成中隐藏，任意一天完成后也不再出现。
11. 首次写入时选择本地文件夹；之后按凌晨 3 点定义的一天更新该文件夹中的待办计划，普通任务记录仍按完成日期保存。设置窗口可查看当天待办、期限事项和按日期分组的普通完成任务，并可删除单条完成记录。

## 状态与文案

| 状态 | 长条 | 对用户的文字 |
| --- | --- | --- |
| 空闲 | 灰色 | 空闲 / 现在想推进什么？ |
| 计划内 | 绿色 | 正在进行 / 剩余约 N 分钟 |
| 到时等待 | 黄色 | 这段时间结束了 |
| 正计时 | 蓝色 | 仍在投入 / 已继续约 N 分钟 |

颜色不是唯一的信息表达。长条始终同时显示状态文字；时间只显示模糊分钟数，绝不显示秒数或“超时”类失败表述。

## 明确不做

不做暂停、番茄钟循环、休息提醒、声音、强制通知、账号、云同步、跨设备同步、AI、屏幕监控、效率评分、排行榜、复杂统计、项目/标签/优先级或多任务并行。
