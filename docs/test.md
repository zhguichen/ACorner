# 测试与验收

## 自动检查

```sh
swift build
swift test
```

单元测试目前覆盖任务记录、待办计划、凌晨 3 点日切和完成度计算的数据边界。计时、浮窗位置、焦点、设置窗口和文件选择器需要在真实 macOS 桌面环境下人工验收。

## 人工验收清单

- 启动后只有灰色可拖动细胶囊条；胶囊条第一行可见状态或当前任务，第二行可见完成度线；重启后胶囊条位置保持不变。
- 悬停或单击胶囊条后卡片展开，移开鼠标不会收起；点击卡片外部后才收起。
- 在胶囊条位于屏幕中央、左边缘和右边缘时展开、收起卡片：展开后胶囊条保持收起前的屏幕位置，并停在卡片外侧；卡片应从胶囊条方向轻微滑出并淡入，收起时向胶囊条方向淡出，不能有抖动。
- 输入框可输入文字；时长只能在数字输入框中键入。确认 1 分钟与 180 分钟均可稳定输入，输入 0 或超过 180 时会限制在有效范围内；开始下一任务时保留上次分钟数。
- 在空闲、计时中、到时等待和收尾卡片上都能看到“设置”按钮；点击后打开设置窗口，设置窗口的“主面板”按钮能把浮窗带回前台。
- 在待办页添加、勾选、取消勾选和删除待办；确认待办与已完成分区正确，已完成项显示勾选状态。
- 在凌晨 3 点前添加至少一条完成和一条未完成待办；3 点后重新展开应用，确认只显示 Daybreak，常规计时、待办页和设置入口不可用。Yesterday’s Echo 应同时显示两类昨天待办，未完成项可“带到今天”。
- 不添加待办直接点击“开始今天”，确认可以进入空闲计时卡；带入或新增待办后确认，今天的长条进度只计算今天的待办，昨天项目不再显示。
- 检查本地目录：每天的待办保存为 `ACornerTodos-yyyy-MM-dd.json`，包含当天确认时间、待办和带入来源；昨天文件在日切后保持不变。
- 在设置的待办页确认“过去未完成”分区按任务脉络只列出一次跨日事项；点击“加入今天”后项目带着同一脉络出现在今天待办中，并立即从过去未完成中隐藏；任意一天完成后该事项也不会出现。来源日期文件不被改写。
- 确认“过去未完成”位于“已完成”之后、“完成记录”之前；点击删除并确认后，同一任务脉络在所有日期的未完成副本都会移除，已完成历史保留。
- 使用现有每日待办文件启动应用；确认文件迁移为带任务脉络的数据，且原始内容保留为同目录 `.backup-v1` 文件。
- 保持应用运行跨过凌晨 3 点；确认无需重启，浮窗下一次展开时自动进入 Daybreak。随后确认今天，确认回到常规空闲卡。
- 当天已确认后保持应用运行至下一个整点；确认面板自动展开但当前应用不被激活。若正在编辑任务内容，确认先显示“整点记录待填写”而不覆盖文字；点击“去记录”后只能写下非空内容并记录，处理前点击卡片外部不会收起面板。
- 在 00:00 至 02:00 完成一条整点记录；确认它写入按凌晨 3 点归属的前一天 `ACornerCheckIns-yyyy-MM-dd.json`，其中含整点时刻、实际记录时刻和文本。让 Mac 睡眠后唤醒，确认不会补出睡眠期间的整点记录，而是等待下一个整点。
- 在“期限”页添加未来、今天和已过期日期的事项；确认它们每天都在 Daybreak 中显示，平时的浮窗长条与任务卡不显示期限事项，并分别标示日期、“今天截止”和“仍待处理”。完成、恢复和删除应立即更新本地 `ACornerDeadlines.json`。
- 添加多个待办后观察胶囊条内进度线；勾选完成越多，进度线越满；未计时时，进度在 1% 至 99% 保持静止，待办操作引起进度变化时才有一组白色圆点向前流动；全部完成后切换为暖色渐变与星点，并且只在刚完成时播放一次流光；正在计时或继续投入时，不论待办完成度均显示冷色进度线，且仅在任务开始后的前三秒流动；没有待办时进度线应为空。
- 点击“开始”后卡片收起，胶囊条变绿；展开后只显示模糊剩余时间，不显示秒数。
- 计时中确认胶囊条第一行显示当前任务标题；长标题应截断，不应挤压完成度线或溢出。
- 在绿色计时中展开卡片，确认可以在较大的记录框里输入文字；收起再展开后文字仍在。
- 用 1 分钟任务验证：计划结束时胶囊条变黄并展开；十秒不操作后变蓝并自动收起。
- 在绿、黄、蓝状态分别验证只有“结束”按钮；点击后进入收尾卡，并继续显示同一个记录框。
- 在收尾卡继续编辑记录，点击“Next Mission”后才保存，并回到下一任务输入。
- 从空闲卡片的“从待办填入”选择一项并走完整结束流程，确认点击“Next Mission”后对应待办自动移入“已完成”分区。
- 完成任务时，首次选择本地文件夹；检查当天的 `ACornerTasks-yyyy-MM-dd.json` 包含名称、计划/实际时间、正计时标记、额外时长、备注和结束方式。
- 在待办窗口检查“完成记录”分区；确认普通完成任务按日期分组显示，且备注可见；删除其中一项后，确认有删除提示、界面立即更新，对应日期的本地文件也不再包含该记录。
- 任务进行中强制退出后重新启动，确认真实时间差决定恢复后的状态。
- 在浅色和深色模式下检查文字可读性，且状态文字不依赖颜色本身。

## 回归触发条件

修改 `TaskSessionModel` 时运行完整状态机清单；修改 `FloatingPanelController` 或 `FloatingSurface` 时运行浮窗、焦点、拖动、进度线和时长输入清单；修改 `RecordStore` 时用一个全新的临时文件夹验证待办 JSON 和每日任务 JSON 的写入和重启恢复。
