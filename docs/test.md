# 测试与验收

## 自动检查

```sh
swift build
swift test
```

单元测试目前覆盖任务记录的数据编码边界。计时、浮窗位置、焦点和文件选择器需要在真实 macOS 桌面环境下人工验收。

## 人工验收清单

- 启动后只有灰色可拖动圆点；重启后圆点位置保持不变。
- 悬停或单击圆点后卡片展开，移开鼠标不会收起；点击卡片外部后才收起。
- 输入框可输入文字；时长只能通过滚轮/触控板滑动选择，开始下一任务时保留上次分钟数。
- 点击“开始”后卡片收起，圆点变绿；展开后只显示模糊剩余时间，不显示秒数。
- 用 1 分钟任务验证：计划结束时圆点变黄并展开；十秒不操作后变蓝并自动收起。
- 在绿、黄、蓝状态分别验证“完成”和“下一个任务”；下一个任务不展示上一任务收尾卡。
- 完成任务时，首次选择本地文件夹；检查 `ACornerTasks.json` 包含名称、计划/实际时间、正计时标记、额外时长、备注和结束方式。
- 在收尾卡填写备注后检查同一 JSON 记录被更新；不填备注时五秒后收起。
- 任务进行中强制退出后重新启动，确认真实时间差决定恢复后的状态。
- 在浅色和深色模式下检查文字可读性，且状态文字不依赖颜色本身。

## 回归触发条件

修改 `TaskSessionModel` 时运行完整状态机清单；修改 `FloatingPanelController` 或 `FloatingSurface` 时运行浮窗、焦点、拖动和滚轮清单；修改 `RecordStore` 时用一个全新的临时文件夹验证 JSON 写入和重启恢复。
