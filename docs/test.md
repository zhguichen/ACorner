# 测试与验收

## 自动检查

```sh
swift build
swift test
```

单元测试目前覆盖任务记录、待办事项和完成度计算的数据边界。计时、浮窗位置、焦点、设置窗口和文件选择器需要在真实 macOS 桌面环境下人工验收。

## 人工验收清单

- 启动后只有灰色可拖动细胶囊条；胶囊条内可见状态文字和完成度线；重启后胶囊条位置保持不变。
- 悬停或单击胶囊条后卡片展开，移开鼠标不会收起；点击卡片外部后才收起。
- 在胶囊条位于屏幕中央、左边缘和右边缘时展开、收起卡片：展开后胶囊条保持收起前的屏幕位置，并停在卡片外侧；卡片应从胶囊条方向轻微滑出并淡入，收起时向胶囊条方向淡出，不能有抖动。
- 输入框可输入文字；时长只能在数字输入框中键入。确认 1 分钟与 180 分钟均可稳定输入，输入 0 或超过 180 时会限制在有效范围内；开始下一任务时保留上次分钟数。
- 点击空闲卡片的“待办”按钮打开待办窗口；在待办页添加、勾选、取消勾选和删除待办；确认待办与已完成分区正确，已完成项显示勾选状态。
- 添加多个待办后观察胶囊条内进度线；勾选完成越多，进度线越满；没有待办时进度线应为空。
- 点击“开始”后卡片收起，胶囊条变绿；展开后只显示模糊剩余时间，不显示秒数。
- 在绿色计时中展开卡片，确认可以在较大的记录框里输入文字；收起再展开后文字仍在。
- 用 1 分钟任务验证：计划结束时胶囊条变黄并展开；十秒不操作后变蓝并自动收起。
- 在绿、黄、蓝状态分别验证只有“结束”按钮；点击后进入收尾卡，并继续显示同一个记录框。
- 在收尾卡继续编辑记录，点击“Next Mission”后才保存，并回到下一任务输入。
- 从空闲卡片的“从待办填入”选择一项并走完整结束流程，确认点击“Next Mission”后对应待办自动移入“已完成”分区。
- 完成任务时，首次选择本地文件夹；检查 `ACornerTasks.json` 包含名称、计划/实际时间、正计时标记、额外时长、备注和结束方式。
- 在待办窗口检查“完成记录”分区；确认普通完成任务也会显示，且备注可见。
- 检查同目录下新增 `ACornerTodos.json`；待办标题、创建时间和完成时间应与界面一致。
- 任务进行中强制退出后重新启动，确认真实时间差决定恢复后的状态。
- 在浅色和深色模式下检查文字可读性，且状态文字不依赖颜色本身。

## 回归触发条件

修改 `TaskSessionModel` 时运行完整状态机清单；修改 `FloatingPanelController` 或 `FloatingSurface` 时运行浮窗、焦点、拖动、进度线和时长输入清单；修改 `RecordStore` 时用一个全新的临时文件夹验证两个 JSON 文件的写入和重启恢复。
